## Персонаж
Всего `204` персонажа, отсортированных по `id`.

#### Схема персонажа

| Ключ           | Тип    | Описание                                            |
|----------------|--------|-----------------------------------------------------|
| id             | Int    | Идентификатор персонажа                             |,
| name           | String | Имя персонажа                                       |,
| species        | String | Вид персонажа                                       |,
| type           | String | Тип или подвид персонажа                            |,
| status         | String | Статус персонажа                                    |,
| occupation     | String | Род деятельности персонажа                          |,
| gender         | String | Пол персонажа                                       |,
| weapons        | Array  | Список оружия, которым обладает персонаж            |,
| bandanna_color | String | Цвет повязки персонажа(необязательное поле)         |,
| affiliation    | String | Принадлежность персонажа к организациям             |,
| image_cover    | String | Ссылка на изображение персонажа                     |,
| url            | String | Ссылка на собственный конечный URL-адреса персонажа |,
| createdAt      | String | Время, когда персонаж был создан в базе данных      |
| location       | Object | Имя ссылки на адрес местоположения персонажа        |,
| episodes       | Array  | Список эпизодов, в которых появился этот персонаж   |,

#### Получить всех персонажей
Вы можете получить доступ к списку персонажей путём добавления  `/character` в URL-адрес.
```json

    "count": 280,
    "pages": 10,
```

#### Получить одного персонажа
Для получения одного персонажа необходимо произвести добавление в URL-адрес, `id` персонажа, в качестве параметра: `/character/1`

```json
  "name": "Donatello",
  "species": "Turtle",
```

#### Фильтрация персонажей
Вы также можете включить фильтры в URL-адрес, добавив дополнительные параметры запроса. Чтобы начать фильтрацию, добавьте знак  `?`, за которым следует запрос `<query>=<value>`.
Если вы хотите связать несколько запросов в одном вызове, используйте знак  `&`, за которым следует запрос.

Например, если вы хотите проверить, сколько существует живых персонажей просто добавьте `?name=character&status=alive` в URL-адрес.

**Доступные параметры:**

* `name`: фильтрация по заданному имени;
* `status`: фильтрация по заданному статусу (жив, мертв или неизвестно);
* `species`: фильтрация по заданному виду;
* `type`: фильтрация по заданному типу;
* `gender`: фильтрация по заданному полу (женский, мужской, бесполый или неизвестно);

```json
{
  "info": {
    "count": 20,
    "pages": 2,
    "next": "https://ninjaturtlesapi.com/api/character/?page=2&name=character&status=alive",
    "prev": null
  },

      "species": "Mutant rat",

      "status": "Alive",
      "locationId": 3,
```
